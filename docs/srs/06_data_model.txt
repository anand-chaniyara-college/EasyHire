6.2.1 User Entity
--------------------------------------------------------------------------------

Table Name: user

┌────────────────────┬─────────────┬──────────┬───────────────────────────────┐
│ COLUMN             │ TYPE        │ NULLABLE │ CONSTRAINTS                   │
├────────────────────┼─────────────┼──────────┼───────────────────────────────┤
│ id                 │ INT         │ NO       │ PRIMARY KEY, AUTO_INCREMENT   │
│ name               │ VARCHAR(50) │ NO       │                               │
│ email              │ VARCHAR(40) │ NO       │                               │
│ password           │ VARCHAR(255)│ NO       │ BCrypt encrypted              │
│ role               │ VARCHAR(30) │ NO       │ Enum-like string              │
│ status             │ BOOLEAN     │ NO       │ true=active, false=inactive   │
│ department_id      │ INT         │ YES      │ FOREIGN KEY -> department     │
└────────────────────┴─────────────┴──────────┴───────────────────────────────┘

Valid Roles: Default-Admin, Admin, Senior-HR, Junior-HR, Department-head, Interviewer

Relationships:
- ManyToOne: department_id -> Department
- OneToMany: mailHistories -> MailHistory
